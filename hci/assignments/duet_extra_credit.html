Extra Credit: Duet

  • Group size: Individual
  • DUE: Thursday, March 31. 11:59PM.

Overview

For 10 points of extra credit on any design sprint, you may complete the following assignment.

Duet

Duet is a product developed by the startup company Intelus.AI. You spoke earlier in the semester with the CPO, Riham Mansour. You will need to sign up for Duet for free. The system is designed to help domain experts (who may not know much about machine learning) to train machine learning models. Machine learning non-experts have very different user needs than those who have prior knowledge of machine learning. In this extra credit assignment, you will be using Duet and giving feedback about the user experience.

Data

There are 3 datasets already available in the platform when you create a new model in the samples library. The three datasets are

  • Enron email dataset
  • ODP (Open Directory Project)â€”a set of webpages about multiple classes
  • Customer support dataset that has a set of Alexa utterances where users are inquiring or complaining about their online shopping
Use these examples only to gain familiarity with the system. These are examples, but your extra credit assignment should not use these datasets.

There are additional publicly available datasets you can use by uploading the dataset in Duet to build other scenarios. Examples of these datasets are product reviews for sentiment analysis scenarios, airline reviews, hotel reviews, restaurant reviews, Reuters dataset, BBC news, meeting notes.

Your Task

There are three main questions we want answered:

  1. How easy is the onboarding experience and where students will be blocked?
  2. Do students understand the teaching loop the way it is offered in the platform today? Can they select the right features to fix conflicts?
  3. What applications can they build where they will need to build a classification model on Duet using any of the datasets we provide whether on the platform or separately?

Deliverables

The output from this assignment should be a PDF report. In terms of length, it should be the approximate equivalent of a 5-10 minute read. Submit the PDF on Canvas. You should answer the three questions listed above. For this assignment, I would expect that to entail a minimum of the following:

  • description of how you used the system to address the question
  • critique of the usable features and areas for improvement of the system
  • justification for that critique, using the language of user-centered design and evaluation
  • accompanying annotated screenshots of the interface
